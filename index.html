Equipo Agile (Srum)

Que es un equipo Agile?

Es un grupo de personas con un fin en comun, que perimite la gestin de proyectos de forma agil, ellos pueden auto organizarse mientras abordan problemas, pueden aprender mediante experiencias y puedes recapacitar sobre sos logros y errores para una mejora continua. Aporta a los trabajadores un conjunto de herramientas, reunuines y funciones que facilitan la estructuracion, coordinacion y gestion del trabajo.

Ceremonias

  • Daily meeting

  • Sprint Plannign

  • Planificacion tactica

  • Refinamiento

  • Sprint Review

  • Retrospectiva

Roles

Product Owner

Esta encorgado de optimizar y maximizar el valor del producto, tambien se encarga de gestionar el Product Backlog. Es fundamental que el PO(Product Owner) tenga los privilegios y los permisos necesarios para tomar cualquier desicion respecto al producto

Bussiness Analyst

Es la persona encargada de entender el modelo de negocio, el plan estrategico y de operaciones de la organizacion y traducirlo en requerimientos tegnicos y funcionales concretos

SCRUM Master

Lider que ayuda al equipo y a la organizacion a utilizar la metodologia SCRUM, se focaliza en la parte de negocio y sus fondos

Equipo Tecnico

El euqipo tecnico esta formado por integrantes especializador los cuales son:

  • Desarrollador FrontEnd
  • Desarrollador BackEnd
  • Administrador de Base de Datos
  • Tester QA Manual
  • Seguridad informatica

Desarrollador ForntEnd

Es el encargado de programar el navegador de una web para que una web sea atractiva para el usuario y a la vez funcione correctamente

Desarrollador BackEnd

Es el encargado de trabajar con el servidor y de la arquitectura interna de una pagina web

Administrador de Base de Datos

Es el encargado de Crear, Manipular y mantener las bases de datos relacionadas a la pagina web

Tester QA Manual

Se encarga de buscar problemas y errores en la pagina web o aplicacion antes de que esta salga al mercado, tambien ayuda a los desarrolladores a buscar una solucion. Las pruebas que un Tester QA Manual puede hacer son de funcionalidad, usabilidad, rendimiento y compatibilidad. El trabajo de un Tester QA Manual es importante ya que ayudan a garantizar la calidad de la pagina y/o aplicacion y que esta sea segura, confiable y facil de usar para los usuarios. Para ser un Tester QA Manual se requieren tanto habilidades tecnicas como no tecnicas; las habilidades tecnicas incluyen conocimiento en programacion, base da datos y sistemas operativos, y las no tecnicas incliyen habilidades de comunicacion, analitica y habilidades para trabajar en grupo

Seguridad informatica

Se encarga de buscar y solucionar cualquier vulnerabilidad de la pagina o aplicacion ante ataques maliciosos
